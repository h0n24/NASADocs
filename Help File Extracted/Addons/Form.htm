Form

  • Forms define how an addon appear in the game and are consist of multiple window controls that output information and/or allow players to interact with the addon.
  • Forms are defined in xml files and each xml form files can contain multiple forms to be loaded with Lua
  • Also contains the Event mappings that allows the Lua code to respond to user interaction with the forms.
  • Layout and properties of the window controls can be edited in the Form Editor in Houston
  • Besides the typical windows controls, there are also a set of game-specific controls that do game specific functions.

Example:

MyTestAddon.xml 

<Form Class="Window" LAnchorPoint="0" LAnchorOffset="0" …
    <Control Class="Window" LAnchorPoint="0" LAnchorOffset…
    <Event Name="MouseButtonUp"  Function="OnListItem… 
</Form>
