Form Editor

A proprietary visual .xml editor that allows the user to define forms, controls, and their properties.

Primary interactions:

  • The form list in the left list pane contains all forms present within the open file with toggles to show or hide their controls.
  • The canvas area is used for the creation, modification, and selection of forms and controls.
  • The borders on the canvas represent the screen space as it will appear in-game.

Top Control Panel

The primary controls for creating, ordering, and aligning forms are found along the top of the Houston interface. Note: Using the window creation buttons will add a form to the canvas with any class-specific properties already defined – this eliminates the guesswork of adding the necessary attributes to a basic window form.

Creation Buttons and Classes:

  • Create Window
  • Create Button
  • Create Text Edit Box
  • Create Combo Box
  • Create Grid Window
  • Create Progress Bar
  • Create Slider Bar
  • Create Tree Control
  • Create Tab Window
  • Create Game-Specific Control

List Order Controls:

  • Delete Form/Control: Clicking this button deletes the currently-highlighted form or control and all of its children.
  • Up: Moves the highlighted control up one layer in the form list, effectively lowering its position in the draw order.
  • Down: Moves the highlighted control down one layer in the form list, placing it higher in the draw order.

Align Window Controls:

  • Align Left Edges: Aligns multiple selected windows on the left.
  • Align Right edges: Aligns multiple selected windows on the right.
  • Align Top Edges: Aligns multiple selected windows by their top edge.
  • Align Bottom Edges: Aligns multiple selected windows by their bottom edge.
  • Center Vertically: Centers all selected windows vertically within their parent.
  • Center Horizontally: Centers all selected windows horizontally within their parent.
  • Distribute Horizontally: Evenly spaces out three or more selected windows (horizontally).
  • Distribute Vertically: Evenly spaces out three or more selected windows (vertically).
  • Match Width: Sets all selected windows to the same width as the highest in the form list.
  • Match Height: Sets all selected windows to the same height as the highest in the form list.
  • Match Size: Sets all selected windows to the same height and width as the highest in the form list. 

Canvas View Controls

The canvas area of Houston is used for defining the position and sizes of forms and controls. It contains a white outline meant to represent the screen space when the game is run – as the client can be resized, this simply a guide and not a 1:1 representation of the screen resolution. The margins of the screen space border can be adjusted under View > Margins; in addition, the following keyboard and mouse inputs are available for altering the canvas view:

  • Mouse Wheel Up: Zoom in.
  • Mouse Wheel Down: Zoom out.
  • Mouse Wheel Click + Drag: Pan across the canvas.
  • Spacebar: Resets the canvas view.

Selecting Forms and Controls

Forms and controls can be selecting by clicking directly on them, clicking them in the form list, or by click-dragging to create a marquee which will select all forms/controls within it.

Form List

Appearing on the left side of the screen is the form list – a tree listing of all forms and the children (controls) they contain. Controls can have children, as well, though it’s inadvisable to layer further down than the grandchild level.

Controls are listed in their approximate draw order, with the lowest level (drawn first) at the top and the highest (drawn last) level at the bottom. The up and down buttons located within the top control panel will shift controls higher or lower amongst controls with the same parent.

Includes

“Includes” are sprite, template, and font files that are referenced to supply their contents to one or more .xml form files. Templates and fonts rarely need to be changed when creating standard UI components; some sprite files are referenced for all form files (including the BaseKit and ClientSprites), though newly-created sprites will have to be added to properly appear. To accomplish this, simply access Files > Add  and select the necessary file.
